# Oracle Archive Redo Log Size Monitoring Guide

## Overview

Oracle Archive Redo Logs are copies of online redo logs that are created when the database is running in **ARCHIVELOG** mode. These files are essential for database recovery, point-in-time recovery (PITR), RMAN backups, and Oracle Data Guard.

The `V$ARCHIVED_LOG` dynamic performance view stores metadata about every archived redo log generated by the database. By querying this view, DBAs can monitor archive log generation, determine storage usage, and identify trends in redo generation.

---

# View Archive Log Size

The following query lists all archived redo logs along with their size in megabytes.

```sql
SELECT
    THREAD#,
    SEQUENCE#,
    FIRST_TIME,
    COMPLETION_TIME,
    BLOCKS,
    BLOCK_SIZE,
    BLOCKS * BLOCK_SIZE / 1024 / 1024 AS SIZE_MB
FROM V$ARCHIVED_LOG
ORDER BY COMPLETION_TIME DESC;
```

### Column Description

| Column | Description |
|---------|-------------|
| THREAD# | Redo thread number (important in RAC environments) |
| SEQUENCE# | Archive log sequence number |
| FIRST_TIME | Time when redo generation began |
| COMPLETION_TIME | Time when archiving completed |
| BLOCKS | Number of blocks in the archive log |
| BLOCK_SIZE | Size of each block (bytes) |
| SIZE_MB | Calculated archive log size in MB |

# Calculate Daily Archive Log Generation

To determine the total amount of archived redo generated each day, use the following query.

```sql
SELECT
    TO_CHAR(COMPLETION_TIME, 'YYYY-MM-DD') AS ARCHIVELOG_DATE,
    SUM(BLOCKS * BLOCK_SIZE) / 1024 / 1024 AS SIZE_MB
FROM V$ARCHIVED_LOG
GROUP BY TO_CHAR(COMPLETION_TIME, 'YYYY-MM-DD')
ORDER BY ARCHIVELOG_DATE DESC;
```

# View Archive Logs Generated in the Last 24 Hours

To display only the archive logs created during the previous 24 hours:

```sql
SELECT
    THREAD#,
    SEQUENCE#,
    FIRST_TIME,
    COMPLETION_TIME,
    BLOCKS,
    BLOCK_SIZE,
    BLOCKS * BLOCK_SIZE / 1024 / 1024 AS SIZE_MB
FROM V$ARCHIVED_LOG
WHERE COMPLETION_TIME >= SYSDATE - 1
ORDER BY COMPLETION_TIME DESC;
```

# Practical Use Cases

- Monitor archive log growth.
- Estimate storage required for archived logs.
- Verify archive log generation after backups.
- Analyze redo generation trends.
- Support capacity planning for the Fast Recovery Area (FRA).
- Troubleshoot excessive archive log generation.

---

# Common Issues

| Issue | Possible Cause | Recommended Action |
|-------|----------------|--------------------|
| Archive destination full | Insufficient FRA or filesystem space | Increase storage or delete obsolete archive logs |
| Excessive archive log generation | High DML activity, batch jobs, index rebuilds | Review workload and archive generation trends |
| RMAN backup failure | Archive destination out of space | Free space or expand the FRA |
| Slow recovery | Missing archived logs | Verify archive log backups and retention policy |
